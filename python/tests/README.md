# rapidfem test suite

Two pillars. Everything is validated against **symbolic ground truth or
closed-form physics** — never against another solver.

```
tests/
  harness/           build→solve helpers + analytical reference library
  geometries/        one module per physics phenomenon (the breadth suite)
  test_renormalization.py   pure-math unit tests (S-param renormalization)
```

The kernel goldens live next to their crate (`crates/*/tests/*_golden_test.rs`)
and are generated by the sympy scripts in `derivations/` — see below.

## Pillar 1 — sympy kernel goldens (the core)

Each numerical kernel routine is pinned **entrywise** to an independent
symbolic derivation. A generator in `derivations/` computes the exact ground
truth and writes a Rust golden test:

| generator (`derivations/…`) | pins |
|---|---|
| `nedelec2/emit_element_golden.py` | R2 element stiffness/mass |
| `nedelec2/emit_coefficients_test.py` | barycentric integral table |
| `nedelec2/emit_tri_mass_golden.py` | surface Robin mass matrix |
| `nedelec2/emit_interp_golden.py` | field/curl reconstruction |
| `waveguide/emit_waveguide_golden.py` | modal β / Z / field |
| `materials/emit_debye_golden.py` | Debye permittivity |
| `materials/emit_material_golden.py` | PML stretch + lossy tensor |
| `td_dg/emit_dg_basis_golden.py` | DG mass + differentiation matrices |
| `td_dg/emit_rhs_curl_golden.py` | DG physical curl operator |
| `td_dg/emit_propagator_golden.py` | Krylov/ETD exp(hA) propagator |

Plus hand-written analytical goldens: `port_eigen_cutoff_test.rs` (Bessel/
sinusoid cutoffs) and `lserk4_convergence_test.rs` (4th-order convergence).

Run: `make test-rust`. Regenerate after a kernel change: `make gen-goldens`
(the golden `.rs` files are marked `GENERATED … do not edit`).

## Pillar 2 — phenomenon geometries (the breadth)

Each `geometries/test_<name>.py` builds a parametric geometry, solves through
`harness.case`, and asserts an extracted quantity against a closed form in
`harness.references` or a conservation law. Every solve is gated under the
**< 100 000 DOF** budget so the suite runs on a laptop.

Phenomena covered: waveguide S-params & dispersion, TEM lines (coax/stripline/
CPW Z₀), cavity & dielectric-resonator modes, skin effect (surface & volume
conductors), proximity effect, dielectric/conductor/Debye/Drude loss &
dispersion, PML & ABC absorption, far-field radiation, energy conservation,
and **scale invariance**.

Run: `make test-py` (full, minutes) or a single file with
`cd python && pytest tests/geometries/test_<name>.py -q`.

## Adding a geometry test

Copy `geometries/test_wr90_waveguide.py` (the exemplar) and adapt:

```python
import numpy as np, pytest, rapidfem as rf
from harness import case, references as ref

@pytest.mark.slow
@case.phenomenon
def test_my_structure():
    g = case.geometry(maxh=rf.lambda_maxh(f_max=12e9))
    air = g.box(...); rf.RectWaveguidePort(...)      # PEC is the default on
    prob, res = case.sweep(g, np.linspace(8e9, 12e9, 5))   # unassigned walls
    assert np.abs(res.sparams[:, 1, 0]).min() > 0.98
```

Rules of thumb:
- Use `harness.case` helpers (`sweep`, `eigenmodes`) — they enforce the DOF
  budget. Keep `maxh` coarse enough that `prob.n_dofs < 100_000`.
- Assert against a closed form in `harness.references` (add shared formulas
  there; keep one-off formulas local to your module to avoid merge conflicts).
- Prefer gauge-free quantities (phase *slope*, conservation, ratios) over
  convention-laden absolutes — see the exemplar's phase-slope assertion.
- Unassigned exterior faces default to **PEC**; use `rf.PMC`/`rf.ABC`/`rf.PML`
  for open or magnetic-wall boundaries.
- Mark `@pytest.mark.slow` (any real solve) and `@case.phenomenon`.

## CI

`.github/workflows/ci.yml`: the `rust` and `python-fast` jobs (kernel goldens +
renormalization math) run on every push/PR; the full `python-full` physics
suite runs on master pushes, nightly, and manual dispatch.
